Creating a Course

In order to create a new course, you will need a configuration file and at least one content file.

Configuration File (config.yaml)

The file named config.yaml will contain all the administrative details of your course. To modify these settings to your choosing, follow the format below:

SettingName: SettingOption

Below is an exhaustive list of all the currently available configuration settings.

SettingName

Usage

Options

CourseTitle

Sets the title of your course

Accepts any text

FirstQuarterDay

Sets the first day of the course

Accepts any date in YYYY-MM-DD format

LastBeforeBreak

Sets the last day of class before a break

Accepts any date in YYYY-MM-DD format

FirstAfterBreak

Sets the first day of class after a break

Accepts any date in YYYY-MM-DD format

LastQuarterDay

Sets the last day of the semester

Accepts any date in YYYY-MM-DD format

ClassOnWeekDays

Sets the days of the week your class will have sessions

For all days of the week: mtwrf

For all days except Thursday, for example: mtwf

ShowPastSessions

Sets how many past sessions will be shown.

Accepts any whole number 0 or greater

PublicErrorMessages

Shows or hides compiler error messages.

Show error messages: true

Hide error messages: false

TimeZone

Sets the time zone that all dates and times will be relative to.

List of supported time zones found here.

DefaultView

Sets the default layout of the course

Current Layouts: List, Table

MenuLinks

Sets the links found at the top of the course page

Each link consists of a MenuLinkName and MenuLinkURL. See example below for proper usage

Content File (content.md)

Files named content.md contain the information to be displayed on a course page. The file is written in Markdown and follows the same formatting rules, found here. Your content file can contain any information you want and is perfect for assignments and tests. However, you will likely want at least one calendar page per course.

Calendar Page

To create a calendar page, start the content.md file with \calendar as the first line. This will tell the CourseUp system that this file should be formatted as a calendar page.

The rest of the calendar page should be composed of various Session objects. The first Session object corresponds to the first day the class meets, as specified by the FirstQuarterDay configuration option. Each subsequent Session object corresponds with the appropriate successive class day--no need to specify specific dates.

Each Session object can have any number of line items, denoted with an asterisk (*). Each line item can contain anything you’d like, from links to other course pages to assignment details.

Each line item can have a specified due date. To assign a due date, end the line with due +#, where # is the number of sessions until the item is due. For example, if your line item is to be due the next session, end the line with due +1. To specify a specific time it is due, write it as such: due +1.17:00. This means the line item is due the date of the next session at 5:00PM. You can also specify a specific date by writing the following: due YYYY-MM-DD, and you can still specify a time with this format; for example, due 2020-05-18.23:55 means the line item is due on May 18, 2020 at 11:55PM.

Adding a new Schedule Layout

Note: This is only a feasible option for those with coding experience.

To add a new view, a class must be created that implements the CalendarView interface. Create a new php a file in the CourseUp/ directory with the following starting code:

<?php

require_once('CalendarView.php');

class [YourName]View implements CalendarView {

    function parseCalendar() {

        // your code here

    }

}

?>

It is highly recommended to take advantage of the Calendar and Config instances where all session information is stored. For a better idea of how to use them, reference the code for the currently supported views: List (CourseUp/ListView.php) and Table (CourseUp/TableView.php).
